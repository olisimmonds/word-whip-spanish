# Standards

## Code Quality
- Modular, readable, maintainable
- Clear function structure
- No unnecessary complexity
- TypeScript types defined in shared locations, not duplicated
- Hooks and utilities extracted from components

## Performance
- Efficient data handling
- Avoid redundant computation
- Memoisation where appropriate

## UI/UX
- Clean, modern, consistent design
- Good spacing and layout
- Clear user feedback (button labels match intent)
- Accessible (aria labels, semantic HTML)

## Testing
- Core functionality covered
- Edge cases handled

## General
- Production-ready quality
- Clear documentation
- No duplicate data entries
- localStorage used for user preference persistence
